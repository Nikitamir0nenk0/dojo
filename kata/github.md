# github

## Concepts

### Repository
Папка с историей изменений. Git отслеживает каждое изменение файла внутри `.git/`. GitHub — облако для хранения и доступа. Без `.git/` — просто файлы, не репозиторий.

### Commit
Сохранённый снапшот изменений с метаданными: автор, время, сообщение, SHA-хеш. `git add` → staging area → `git commit` → коммит. Незакоммиченные изменения в коммит не попадают.

Состояния файла:
- **untracked** — git видит, но никогда не трекал
- **modified** — git трекает, видит изменения, не в staging
- **staged** — добавлен через `git add`, войдёт в следующий коммит

### Branch
Указатель на коммит. Не копия файлов. Создать ветку = создать новый указатель (почти бесплатно). `main` — такая же ветка. Коммиты на одной ветке недоступны с другой до merge/rebase.

### Push / Pull / Remote
Remote — ссылка на удалённый репозиторий (`origin` = GitHub). `push` отправляет коммиты на remote. `pull` = `fetch` + `merge`. Если remote содержит коммиты, которых нет локально — push отклоняется (non-fast-forward). Нужен `pull` → разрешить конфликты → `push`.

## Questions

Q: Ты удалил `.git/`. Что произошло?
A: Локальный репозиторий уничтожен. Git-команды не работают. Файлы остались, GitHub не затронут.

Q: Коллега прислал zip без `.git/`. Что ты получил?
A: Снапшот файлов без истории коммитов, веток, авторства.

Q: `git add` сделан для 2 из 3 файлов, потом `git commit`. Где третий?
A: В состоянии modified или untracked — git его видит, но в коммит он не попал.

Q: Сделал 2 коммита на `feature`, вернулся на `main` — изменений нет. Почему?
A: `main` — отдельный указатель, он не сдвигался. Коммиты живут на `feature`.

Q: Коллега запушил в `main`, ты тоже пытаешься пушить — отказ. Почему?
A: Non-fast-forward: remote содержит коммиты, которых нет локально. Нужен `git pull` сначала.

## Notes

Zip с `.git/` внутри — полноценный репозиторий. GitHub → Download ZIP намеренно исключает `.git/`.
