# git-basics

## Concepts

### 1. Что такое Git
Git — система контроля версий. Запоминает каждое изменение в файлах, позволяет вернуться к любой предыдущей версии.
- **Репозиторий (repo)** — папка, за которой следит Git
- **Коммит (commit)** — «снимок» состояния файлов, уникальный ID, сообщение
- **Ветка (branch)** — независимая линия разработки; по умолчанию `main`

### 2. GitHub vs Git
- **Git** — программа на локальном компьютере, отслеживает изменения
- **GitHub** — облако для хранения репозиториев (как Google Drive, но для Git)
- Потеря ноутбука ≠ потеря репо на GitHub

### 3. Структура репозитория
- Файлы/папки, Commits, Branches, README.md, Issues, Pull Requests
- История коммитов: в ветке → вкладка Commits

### 4. Рабочий цикл
`pull → edit → stage → commit → push`
- pull перед push обязателен — иначе Git откажет (diverged history)

### 5. Сообщения коммитов
Формат: `тип: описание что изменилось`
Типы: `fix`, `add`, `update`, `remove`, `refactor`
Суть, не имя файла:
- ✅ `fix: некорректный расчёт налога`
- ❌ `fix budget.py`
